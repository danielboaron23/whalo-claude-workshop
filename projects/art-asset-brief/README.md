# 📂 מארגן נכסים + בריף ויזואלי — Asset Brief

כלי לדוגמה לצוות **Art** · נבנה ב-Claude Code · חלק מסדנת Whalo.

## מה זה עושה
(1) מארגן את ספריית הנכסים — קטלוג עם תגיות וסינון לפי קטגוריה.
(2) מייצר בריף ויזואלי לנכס חדש, עקבי עם הסגנון הקיים.

## מבנה
```
art-asset-brief/
├── GOAL.md · README.md · STEPS.md
├── data/assets.json            ← קטלוג הנכסים (מדומה)
├── app/index.html              ← הקטלוג + הבריף
└── .claude/skills/asset-brief/SKILL.md
```

## איך מריצים
פותחים את `app/index.html` בדפדפן. רץ offline.

## 🔌 חיבור (Google Drive)
בעבודה אמיתית הנכסים יושבים ב-Drive. אפשר לבקש: *"סרוק את תיקיית הנכסים ב-Drive,
תייג כל נכס, וצור בריף לנכס חדש שמתאים לסגנון."*

## הדפוס שלוקחים
**ספרייה קיימת → סדר + תוצר חדש שמתאים לה.** עקביות ויזואלית בלי מאמץ ידני.
