# ▶️ הרצה חיה — מסנן קורות חיים

ארגז חול מוכן להדגמה מול הקהל. שתי דרכים — התחילו ב-א' (בטוח), סיימו ב-ב' (וואו).

**מה יש בתיקייה:**
```
demo/
├── job-description.md          ← המשרה
├── candidates/                 ← 4 קורות חיים (מצוין / בינוני / חלש / דומיין-חזק-קוד-חלש)
│   ├── cv-01-noa-barak.txt
│   ├── cv-02-yoav-peled.txt
│   ├── cv-03-shira-dahan.txt
│   └── cv-04-eitan-mor.txt
└── candidates-for-notion.csv   ← לייבוא ל-Notion (לדרך ב')
```

---

## דרך א' · על קבצים מקומיים (הכי פשוט)
> מדגים: Claude קורא מספר קבצים אמיתיים ומריץ את ה-Skill.

1. ודאו שאתם בתיקיית הפרויקט (ה-Skill `resume-screener` כבר כאן).
2. כתבו ל-Claude Code:
   > „סנן את כל קורות החיים בתיקייה `demo/candidates/` מול המשרה ב-`demo/job-description.md`,
   > דרג אותם עם ה-Skill resume-screener, והצג טבלה מדורגת.”
3. Claude קורא את 4 הקבצים, מדרג, ומחזיר טבלה עם ציון, המלצה ונימוק. **זה הדמו.**

💡 טיפ במה: התחילו במועמד אחד („תדרג לי את `cv-01`”), ואז „עכשיו על כולם” — הקפיצה מרשימה.

---

## דרך ב' · עם MCP / Connector (Notion) — „ככה זה בעבודה אמיתית”
> מדגים: Claude קורא מ-Notion אמיתי, מדרג, וכותב את התוצאות בחזרה — בלי token, בלי קוד.

1. ייבאו את `candidates-for-notion.csv` למסד **Candidates** ב-Notion
   (או השתמשו במסד שכבר קיים — Claude יכול גם ליצור אותו).
2. כתבו ל-Claude Code:
   > „דרך ה-connector של Notion, קרא את המועמדים ממסד Candidates, דרג אותם מול המשרה,
   > וכתוב לכל אחד Score, Recommendation ו-Reasoning בחזרה.”
3. עברו לטאב של **Notion** → הציונים מתעדכנים חי בכל רשומה. **זה הוואו.**

---

## מה הקהל לוקח מזה
- **דרך א'** מראה את המנגנון: Claude + קבצים + Skill → תוצאה.
- **דרך ב'** מראה את הערך האמיתי: אותו Skill בדיוק, אבל על הכלים האמיתיים שלהם (Notion),
  וכותב בחזרה — בלי שום הקמה.
