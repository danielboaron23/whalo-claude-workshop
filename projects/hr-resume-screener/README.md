# 📂 מסנן קורות חיים — Resume Screener

כלי לדוגמה לצוות **HR** · נבנה כולו ב-Claude Code · חלק מסדנת Whalo.

## מה זה עושה
מקבל תיאור משרה + קורות חיים, ומחזיר **shortlist מדורג** עם ציון ונימוק
לכל מועמד, לפי rubric אחיד ושקוף.

## מבנה הפרויקט
```
hr-resume-screener/
├── GOAL.md                 ← המטרה: מה בונים, למי, ולמה
├── README.md               ← הקובץ הזה
├── STEPS.md                ← מדריך הבנייה שלב-שלב (ה-Lab)
├── data/
│   ├── job-description.md   ← המשרה (input לדוגמה)
│   └── candidates.json      ← 5 מועמדים בדויים
├── app/
│   └── index.html           ← האפליקציה הרצה (פותחים בדפדפן)
└── .claude/skills/resume-screener/
    └── SKILL.md             ← ה-Skill שמדרג את המועמדים
```

## איך מריצים
פותחים את `app/index.html` בדפדפן (לחיצה כפולה). זהו — הכל מוטמע, רץ גם בלי אינטרנט.

## ✨ העלאת קורות חיים (חי)
באפליקציה יש אזור **"העלו קורות חיים לבדיקה"**: גוררים קובץ `.txt/.md` (או מדביקים טקסט),
לוחצים **"בדוק ודרג"**, והמועמד מקבל ציון לפי ה-rubric ונכנס לרשימה המדורגת עם תג "חדש".
קבצים לדוגמה להעלאה נמצאים ב-`data/sample-cvs/` (מצוין / בינוני / חלש).

> הבדיקה בדפדפן היא גרסה מהירה מבוססת-כללים *להדגמה*. הניתוח המעמיק האמיתי נעשה ע״י
> Claude/ה-Skill — פשוט מוסיפים CV ל-`data/` ומבקשים מ-Claude לדרג.

## 🔌 חיבור ל-Notion (connector)
האפליקציה מחוברת למסד מועמדים **„Candidates”** ב-Notion (מדומה ב-`data/notion-candidates.json`):
- **⤓ סנכרן מועמדים מ-Notion** — מושך מועמדים מהפייפליין, מדרג אותם ומכניס לרשימה.
- **⤴ כתוב דירוג בחזרה ל-Notion** — כותב לכל מועמד Score · Recommendation · Reasoning.

> החיבור באפליקציה הוא **דימוי** להדגמה (HTML סטטי לא ניגש ל-Notion בלי הרשאות).
> האינטגרציה האמיתית רצה ב-**Claude Code עם ה-connector של Notion** — ראו `STEPS.md` שלב 5.7.
> אותו דפוס עובד גם עם Airtable / Google Drive / Gmail.

## איך זה עובד (knowledge base)
1. **ה-Skill** (`resume-screener`) מגדיר ל-Claude *rubric*: אילו קריטריונים,
   באילו משקלים, ובאיזה פורמט פלט.
2. **הדאטה** ב-`data/` היא ה-input — המשרה והמועמדים.
3. Claude קורא את שניהם, נותן ציון ונימוק לכל מועמד, ומחזיר טבלה מדורגת.
4. **האפליקציה** מציגה את התוצאה בצורה יפה ואינטראקטיבית.

## למה זה דפוס חשוב
*rubric + מסמכים → פלט מובנה* חוזר כמעט בכל משימת HR:
דירוג מועמדים, השוואת ספקים, ציון משימות, בדיקת התאמה. בנו פעם אחת — השתמשו תמיד.

## רעיונות להרחבה
- להוסיף קריטריון משלכם ל-rubric ולראות איך הדירוג משתנה.
- לבקש מ-Claude לנסח מייל זימון/דחייה אדיב לכל מועמד.
- להזין משרה אמיתית (לא רגישה) משלכם.
