> **דמו חי לסדנה.** ניקח **משימה אחת** ונעשה אותה בשני הכלים, כדי לראות את ההבדל האמיתי —
> לא "מה אפשרי", אלא **תחום העבודה**: עבודת-ידע (Cowork) מול הנדסה/בנייה (Claude Code).
>
> **המשימה:** *"נתח את תוצאות ה-A/B pricing ותגיד לי איזה מחיר לשלוח."*
> **הדאטה:** הקובץ `Reef Rumble — Starter Pack Price A/B Test` (Google Sheet / CSV) — 4 וריאנטים, מספרים גולמיים בלבד.

## 🤝 מסלול א' — Cowork ("קבל את התוצר")

**איפה:** אפליקציית Claude לדסקטופ → מצב **Cowork**.

### מה עושים
1. פותחים את אפליקציית Claude ובוחרים **Cowork**.
2. **גוררים** את ה-CSV (או נותנים גישה לתיקייה שבה הוא יושב).
3. מדביקים את הבקשה:

```
נתח את pricing-ab-results.csv:
לכל וריאנט חשב conversion, ARPU, revenue ומובהקות מול ה-Control.
הדגש את המנצח, והמלץ איזה מחיר לשלוח.
תכין דוח מעוצב בן עמוד — טבלה + שורת המלצה.
```

### מה קורה
- Claude מראה **תוכנית** קצרה לפני שהוא פועל.
- קורא את הקובץ, מחשב הכל מהמספרים הגולמיים, ומחזיר **דוח מוכן** (Doc/Sheet מעוצב).

### התוצר (דוגמה)
| variant | price | conversion | revenue | ARPU |
|---|---|---|---|---|
| A (Control) | $2.99 | 7.20% | $1,291.68 | $0.215 |
| B | $3.99 | 6.51% | $1,560.09 | $0.260 |
| C | $4.99 | 5.89% | $1,761.47 | **$0.294** 🏆 |
| D | $5.99 | 4.80% | $1,725.12 | $0.287 |

> **המלצה:** לשלוח **$4.99 (וריאנט C)** — ה-ARPU הגבוה ביותר; מעבר לזה ($5.99) ה-conversion צונח וההכנסה יורדת.

### ה"וואו" של Cowork
> *"תעשה את זה **כל יום שני בבוקר** אוטומטית, ותשלח לי את הדוח."* — Cowork **מתזמן** את זה. בלי קוד, בלי התקנה.

✅ **מתי לבחור Cowork:** צריך את התשובה **עכשיו**, יפה ומוגשת — בלי להקים כלום.

## 🛠️ מסלול ב' — Claude Code ("בנה את הכלי")

**איפה:** אפליקציית Claude לדסקטופ (או טרמינל/VS Code) → פותחים את **תיקיית הפרויקט**
`projects/monetization-pricing-summary/`.

### מה עושים
1. פותחים את תיקיית הפרויקט ב-Claude Code.
2. מדביקים את הבקשה:

```
בנה Skill בשם pricing-summary:
מקבל CSV של תוצאות A/B ומחשב לכל וריאנט conversion, ARPU, revenue ומובהקות.
מדגיש את המנצח וממליץ איזה מחיר לשלוח. פורמט קבוע: טבלה + שורת המלצה.
הרץ אותו על data/pricing-ab-results.csv, ובנה app/index.html כדשבורד שמציג את התוצאה.
```

### מה קורה
```
אתם ▸ בנה Skill בשם pricing-summary …
Claude ▸ על זה — בונה את הכלי…
        ✓ נוצר .claude/skills/pricing-summary/SKILL.md
        ✓ נוצר app/index.html (דשבורד)
        ✓ הורץ על data/pricing-ab-results.csv
        ✦ הווריאנט המנצח: C ($4.99) · ARPU $0.294
```

### התוצר — לא דוח, אלא **כלי שנשאר**
```
monetization-pricing-summary/
├─ .claude/skills/pricing-summary/
│  ├─ SKILL.md           ← ה"מתכון" — הנוסחאות + הפורמט הקבוע
│  └─ references/        ← הידע: נוסחאות significance, סף החלטה
├─ app/index.html        ← דשבורד שרץ בדפדפן
└─ data/pricing-ab-results.csv
```

### ה"וואו" של Claude Code
- **`git commit`** → כל הצוות מקבל את ה-Skill. בפעם הבאה פשוט אומרים *"run pricing-summary"*.
- מחברים **Mixpanel (MCP)** → הכלי מושך את הניסוי הבא **לבד**, בלי CSV ידני.
- אותו ניתוח **מדויק ועקבי**, על **כל** ניסוי, לתמיד.

✅ **מתי לבחור Claude Code:** תחזרו על זה שוב ושוב, רוצים **דיוק, שליטה ושיתוף** בצוות, או כלי/אפליקציה אמיתית.

## ⚖️ אותה משימה — מה באמת שונה

| | 🤝 Cowork | 🛠️ Claude Code |
|---|---|---|
| **הכנה** | אפס — גוררים קובץ | בונים פעם אחת |
| **קוד?** | לא | Claude כותב — אתם מתארים |
| **התוצר** | דוח/Sheet מוכן (חד-פעמי או מתוזמן) | Skill + אפליקציה שחוזרים ומשותפים |
| **חוזר על עצמו?** | מתזמנים את אותה משימה | בונים כלי שכל הצוות מריץ |
| **מתחבר למקור חי** | connectors באפליקציה | MCP מלא (Mixpanel, git, ועוד) |
| **המתאים ל** | תוצאה מוגשת, מהר, בלי-טכני | מערכת/כלי/אוטומציה הנדסית |

> **השורה התחתונה:** Cowork נותן לך את ה**תוצר**. Claude Code בונה לך את ה**כלי שמייצר את התוצר**.
> מתחילים מ-Cowork (קל ובטוח), ומשדרגים ל-Code כשרוצים יותר כוח, דיוק ושליטה.
